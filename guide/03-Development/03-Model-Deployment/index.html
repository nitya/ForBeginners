    Change Model Deployment¶

    This lab is under active development and testing. Changes are likely.

    By the end of this unit, you will be able to

    • Understand different Azure OpenAI models and their capabilities
    • Configure custom model deployments with appropriate capacity settings
    • Compare model performance and costs for your specific use case

    1. Understanding Azure OpenAI Models¶

    Choosing the right AI model is like selecting the right tool for a job. Each model has different strengths, costs, and capabilities. Think of models as different specialists: some are great at conversation, others excel at analysis, and some are optimized for speed or cost-effectiveness.

    2. Available Model Options¶

    GPT-4o Models: - gpt-4o: Latest flagship model with multimodal capabilities - gpt-4o-mini: Optimized for speed and cost while maintaining quality - Best for: Complex reasoning, content creation, code generation

    GPT-3.5 Models: - gpt-35-turbo: Fast and cost-effective for many tasks - gpt-35-turbo-16k: Extended context window for longer documents - Best for: Simple conversations, basic content generation

    Embedding Models: - text-embedding-ada-002: Standard embedding model - text-embedding-3-small: Newer, more efficient embedding model - text-embedding-3-large: Higher dimensional embeddings for better accuracy

    Model Configuration Parameters¶

    Deployment Capacity: Controls throughput (tokens per minute) - Higher capacity = faster responses but higher cost - Standard capacities: 30, 60, 100, 240 TPM (thousands tokens per minute)

    Model Version: Specific model release version - Latest versions often have improved capabilities - Consider stability vs. newest features

    Deployment SKU: Pricing tier - Standard: Pay-per-use pricing - Provisioned: Reserved capacity for consistent performance

    Cost and Performance Trade-offs¶

    Performance Factors: - Response speed and latency - Context window size (how much text the model can process) - Reasoning and accuracy capabilities

    Cost Factors: - Per-token pricing varies by model - Deployment capacity affects minimum costs - Regional pricing differences

    Lab Time ☑️¶

    Let's customize the model deployment for your AI agent:

    - [X] **Review current model configuration**:

    bash title="" linenums="0" # Navigate to your project cd /workspaces/AZD-for-beginners/workshop/ai-agents-project # Check current model settings azd env get-values | grep -i model azd env get-values | grep -i deploy

    Text Only
    1
    - [X] **Choose your model configuration** (select one option):

    Option A - Upgrade to GPT-4o (Best quality): 

    # Set the primary model to GPT-4o
azd env set AZURE_AI_AGENT_MODEL_NAME "gpt-4o"
azd env set AZURE_AI_AGENT_MODEL_VERSION "2024-08-06"
azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 30

echo "Configured for GPT-4o with standard capacity"

    Option B - Use GPT-4o-mini (Balanced cost/performance): 

    # Set the model to GPT-4o-mini (often the default)
azd env set AZURE_AI_AGENT_MODEL_NAME "gpt-4o-mini"
azd env set AZURE_AI_AGENT_MODEL_VERSION "2024-07-18"
azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 100

echo "Configured for GPT-4o-mini with higher capacity"

    Option C - High-performance configuration: 

    # Set higher capacity for better performance
azd env set AZURE_AI_AGENT_MODEL_NAME "gpt-4o-mini"
azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 240
azd env set AZURE_AI_AGENT_DEPLOYMENT_SKU "Standard"

echo "Configured for high-performance deployment"

APP_URL=$(azd show --query services.api.endpoint -o tsv)

    2. Compare model performance:

    Test these scenarios with your new model:

    Response Quality Test: - Ask the same question you tested before - Compare the depth and accuracy of responses - Note any differences in reasoning capability

    Speed Test: - Ask a complex question that requires knowledge retrieval - Observe response time differences - Note if higher capacity improves speed

    Context Handling Test: - Ask follow-up questions in the same conversation - Test how well the model maintains context - Try longer, more complex queries

    1. Monitor costs and usage: 

      # Check resource group for cost monitoring
RESOURCE_GROUP=$(azd env get-value AZURE_RESOURCE_GROUP)
echo "Monitor costs in Azure Portal for resource group: $RESOURCE_GROUP"

# View current deployment details
azd show --output table

    2. Troubleshoot common issues:

      If deployment fails due to quota: 

      # Reduce capacity or try different region
azd env set AZURE_AI_AGENT_DEPLOYMENT_CAPACITY 30
azd env set AZURE_LOCATION "East US 2"  # Try different region
azd up

      If model isn't working as expected: 

      # Check agent configuration in Azure AI Foundry
azd env get-value AZURE_EXISTING_AIPROJECT_RESOURCE_ID
echo "Verify model deployment in AI Foundry portal"

    Summary¶

    Changing model deployments allows you to optimize your AI agent for your specific needs - whether that's maximizing quality, minimizing cost, or balancing performance. Different models excel at different tasks, and the right choice depends on your use case and budget.

    The key insight is that model selection isn't just about capability - it's about finding the right balance of performance, cost, and reliability for your specific application needs.

    Model Configuration Success Indicators: - ✅ New model deploys without quota errors - ✅ Agent responses reflect the new model's capabilities - ✅ Performance meets your requirements - ✅ Costs align with your budget expectations
